# TCI Renaissance — Layered 2.5D Edition

这是一个独立于 `editions-mvp` 的新版本，用轻量 2.5D 分层素材强化滚动时的景深、主体运动和场景转场。它拥有自己的入口、样式、脚本和生成素材；维护或实验本目录不会要求修改原有 MVP。

## 四层素材结构

每个场景在 `assets/scenes/` 中使用同一组命名规则：

- `*-bg.webp`：低频、轻微模糊的背景层，负责远景和整体色调；
- `*-mid.webp`：软椭圆蒙版的中景层，补充空间层次；
- `*-subject.webp`：对比度和锐度更高的主体层，承担主要视差运动；
- `*-fore.webp`：从画面底部提取的前景层，以更大的位移制造近景速度感。

此外还有两张共享气氛图 `atmosphere-gold.webp`、`atmosphere-data.webp`，以及 `assets/overlays/` 中的尘埃和笔触装饰 SVG。

## 运行时核心

- WebGL2 同时绑定“当前场景 + 下一场景”的 `4 × 2` 张纹理；每个场景先完成背景、中景、主体、前景合成，再进行章节转场，主体不会像 DOM 贴纸一样悬在转场之外。
- 四层使用不同的缩放、滚动位移、鼠标视差和速度惯性，形成轻量 2.5D 景深；全部状态由滚动位置计算，向上滚动会反向恢复。
- 章节边界使用固定约 `1.25` 个视口高度的三阶段交接：旧场景压暗与降饱和、带噪声笔触揭示、新场景提亮落位。
- 透明 WebP 使用预乘 Alpha 上传和合成，降低羽化边缘的暗边；交界处的下一场景末帧与新章节首帧使用完全相同的图层姿态，避免跨界跳动。
- CPU 侧默认只缓存当前和下一场景；接近边界时短暂预取下下场景。远距离 Hash 跳转或弱设备加载期间保留对应海报 fallback，不露黑屏。
- 正文 Beat 的透明度、位移、缩放和模糊由视口进度连续控制；静止后停止 DOM 样式写入，WebGL 待机降到约 30fps。

## 重新生成分层素材

生成脚本依赖 ImageMagick 的 `magick` 命令，并从项目现有的 `assets/` 原图重建全部 WebP 分层素材：

```bash
cd "/Users/alan/work/SZTCI HP/tci-reshape-demo/editions-layered"
./tools/build-layers.sh
```

生成结果会覆盖写入 `editions-layered/assets/scenes/`。如果系统尚未安装 ImageMagick，需要先完成安装再运行脚本。

## 本地运行

从 `tci-reshape-demo` 根目录启动静态服务器：

```bash
cd "/Users/alan/work/SZTCI HP/tci-reshape-demo"
python3 -m http.server 4173
```

然后访问：

```text
http://127.0.0.1:4173/editions-layered/
```

不要直接双击 `index.html` 打开；本地 HTTP 服务能让浏览器以与实际部署更接近的方式加载脚本和素材。

## 轻量软蒙版的局限

当前分层由同一张扁平原图通过椭圆、矩形和高斯模糊软蒙版生成，并不是真正的语义分割或 Blender 2.5D 场景。因此：

- 主体边缘可能携带少量原背景，视差幅度过大时会显露重影；
- 被主体遮挡的背景没有经过完整修补，不能像真实相机一样大范围移动；
- 桌面与移动端共用同一套 16:9 分层，极窄屏幕的裁切自由度有限；
- 人物骨骼、独立灯光、景深遮挡和真实三维旋转不在这个轻量版本的范围内。

为了保持画面稳定，应将主体和前景的位移控制在较小范围。若要继续逼近完整商业制作，应为关键主体准备透明抠图，为背景制作无人物修补底板，并在需要时升级到 Blender、GLB 与独立灯光/相机时间线。
